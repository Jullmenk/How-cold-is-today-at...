Objetivos do Desafio

Este desafio tem como objetivo avaliar desenvolvedores WEB que se inscrevem para a WIT.

Pressupõe-se que o candidato que assumir esse desafio não tenha experiência com React, mas tenha interesse no desenvolvimento frontend e em tecnologias web (CSS, SASS, HTML, JS, TS, etc.).

O desafio busca avaliar a capacidade dos candidatos de aprender uma nova tecnologia e concluir uma tarefa dentro de um prazo definido.

Descrição

Sua tarefa será desenvolver um site responsivo usando uma biblioteca existente, no qual seja possível obter a previsão do tempo para 5 dias de uma cidade inserida pelo usuário.

Em todos os casos, o site deve exibir a temperatura atual. Se não for possível obter a previsão, o usuário deve ser avisado com um erro contextualizado, explicando a razão pela qual não foi possível obter as informações.

Você deve dar atenção especial à interface gráfica (UI) e à experiência do usuário (UX) em três tipos de tela principais: desktop, tablet e celular.

Como fonte de dados, é preferível o uso de APIs no formato JSON.

Desafios Extras (sem ordem específica de importância):

O usuário deve ser capaz de selecionar a unidade de medida de temperatura entre Celsius e Fahrenheit. Ao fazer essa alteração, todas as temperaturas apresentadas devem se adaptar à nova unidade sem um refresh completo da página. (+1)
Apresentar um gráfico com a evolução da temperatura por dia. (+3)
Apresentar um mapa de temperatura centrado na cidade escolhida. (+2)
Métricas de Avaliação

Os parâmetros de avaliação para este desafio, sem ordem específica de importância, são:

Estrutura e qualidade do código;
UI e UX construídas;
Desempenho do site;
Capacidade de cumprir prazos.
Dicas Úteis

Para estilizar os diferentes componentes, sugere-se o uso do Styled Components (https://styled-components.com/).
Para a criação de possíveis formulários, sugere-se o uso do FORMIK (para construção de formulários) + YUP (para validações) [https://formik.org/ e https://github.com/jquense/yup].
Sugere-se que o código produzido seja em React com TypeScript.
Sugere-se que os componentes sejam React Function Components em vez de Class Components, sempre que possível. Dê preferência ao uso de React Hooks.
A API OpenWeatherMap (https://openweathermap.org) fornece todos os dados necessários para a execução do desafio.
Para conseguir fazer requisições para uma API externa, pode ser necessário alterar o arquivo hosts.
